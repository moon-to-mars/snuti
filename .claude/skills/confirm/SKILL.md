# 확인 단계

## 목적
리뷰에서 받은 피드백을 반영해 코드를 수정한다. 리뷰 → 확인의 루프는 사용자가 만족할 때까지 반복된다.

## 절차

### 1. 피드백 반영
사용자의 피드백을 바탕으로 코드를 수정한다.

피드백이 모호하다면 바로 수정하지 말고 먼저 선택지로 확인한다. 예시:

```
어떤 부분을 수정할까요?
1. 로직(동작 방식)을 바꿔줘
2. 코드 스타일(구조, 가독성)을 바꿔줘
3. 둘 다
```

### 2. 수정 내용 요약
수정이 끝나면 어떤 부분을 어떻게 바꿨는지 간단히 요약한다.

```
수정 내용:
- [변경한 부분]: [변경 이유]
```

## 완료 조건
수정 완료 후 → workflow가 `review` 스킬을 다시 발동한다 (리뷰 ↔ 확인 루프)
