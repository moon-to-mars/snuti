# Commit 준비 단계

## 목적
하위 항목 하나의 구현이 완료될 때마다 커밋을 안내한다.
하나의 feature 안에서 하위 항목 수만큼 커밋이 반복될 수 있다.

## 절차

### 1. 변경사항 요약
이번 하위 항목 구현에서 수정/생성된 파일과 핵심 변경 내용을 간단히 정리한다.

```
## 변경사항 요약
- [파일명]: [변경 내용 한 줄 요약]
- [파일명]: [변경 내용 한 줄 요약]
```

### 2. 커밋 전 체크포인트 — 1개씩 확인
아래 항목을 **하나씩** 제시한다. 사용자가 응답하면 다음 항목으로 넘어간다.

항목마다 아래 형식으로 제시한다:
```
(N/3) [항목 내용]
1. 확인 완료
2. 문제 있어 (어떤 문제인지 말해줘)
3. 해당 없음
```

확인 항목:
1. 콘솔 에러 없음
2. 환경변수 노출 없음
3. 하드코딩된 매직 넘버 없음

"문제 있어"를 선택하면 해당 문제를 즉시 처리한 뒤 다음 항목으로 넘어간다.

### 3. checklist.md 하위 항목 업데이트
완료된 하위 항목을 `- [ ]` 에서 `- [x]` 로 변경한다.

### 4. 커밋 안내
커밋 메시지 예시를 제안하고 사용자가 직접 커밋하도록 안내한다.

```
커밋 메시지 예시: feat: [구현한 내용 한 줄 요약]

커밋 후 어떻게 할까요?
1. 커밋 완료, 다음 항목 구현하러 가자
2. 커밋 완료, 이번 feature 다 끝났어
3. 잠깐, 수정할 게 생겼어
```

### 5. 다음 단계 분기
커밋 완료 후 checklist.md에서 현재 feature의 미완료 하위 항목을 확인한다.

- **미완료 하위 항목이 있다** → "다음 하위 항목: [항목명] 을 설계할게요." 라고 안내하고 workflow가 `design` 스킬을 발동한다
- **모든 하위 항목이 완료됐다** → workflow가 `push` 스킬을 발동한다
