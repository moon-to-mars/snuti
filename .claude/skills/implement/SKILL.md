# 구현 단계

## 목적
설계 단계에서 합의한 내용을 실제 코드로 작성한다.

## 핵심 규칙 — 반드시 준수

**하위 항목 하나만 구현하고 멈춘다.**

checklist.md에서 현재 feature의 미완료 하위 항목 중 가장 위에 있는 것 하나만 구현한다. 나머지 항목은 이번 구현 세션에서 절대 건드리지 않는다. 구현이 끝나면 반드시 멈추고 review 스킬로 넘어간다.

예시:
- ❌ 잘못된 방법: 스키마 + Repository + Service + 라우터를 한 번에 구현
- ✅ 올바른 방법: 스키마만 구현 → 멈춤 → review → commit → 다시 implement(Repository만)

## 절차

1. design 단계에서 합의한 설계 요약을 바탕으로 코드를 작성한다
2. 하위 항목 하나만 구현하고 멈춘다
3. 구현이 끝나면 review 스킬로 넘어간다

구현 중 예상치 못한 결정이 필요하다면:
- 사소한 것은 합리적인 판단으로 직접 결정하고 나중에 리뷰에서 설명한다
- 중요한 것은 잠깐 멈추고 사용자에게 물어본다

## 완료 조건
하위 항목 하나의 코드 작성이 끝났을 때 → workflow가 `review` 스킬을 발동한다
